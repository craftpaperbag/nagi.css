なぜ作ったのか

多くのCSSフレームワークは、機能を足すことで成立しています。色、影、角丸、アニメーション、グリッドシステム。けれど、読むためのページに、それらは本当に必要でしょうか。

nagi.css は、引き算で設計されたクラスレスCSSです。<link> を一行貼るだけで、セマンティックに書かれたHTMLが静かに整います。書く側は、正しいタグを選ぶことだけに集中できます。

装飾で補強しないと伝わらない情報は、まだ十分に削られていない。

導入

CDN経由で、一行。

<link rel="stylesheet" href="https://koharunagi.github.io/nagi.css/nagi.css">

ローカルに置くなら、nagi.css をダウンロードして読み込んでください。

あとは、クラスを書かずにセマンティックなHTMLを書くだけです。<article>、<section>、<header>、<nav>、<footer>、見出し、段落、リスト。普通に書けば、普通に整います。

要素

見出し

weight は 500 までしか使いません。大きさと余白だけで階層をつくります。

段落とリンク

段落の行長は 60ch（日本語で約38文字）、読みやすさとページとしての存在感のバランスを取った幅です。リンクは太字＋下線で明確に、ホバーで下線が太くなります。

番号付きリスト

  1. 壁打ち相手として使う
  2. 長文を要約させる
  3. 文章の推敲を頼む

箇条書き

  • 色は文字と罫線のグレーだけ
  • カードで囲わず、罫線と余白で区切る
  • アニメーションは置かない

定義リスト

classless
クラスを書かず、タグに直接スタイルを当てる設計。
semantic
意味に基づいたタグ選択で、構造と見た目を一致させる。
measure
読みやすい行長。nagi.cssの既定値は日本語向けに 40ch。

引用

道具は、自分の信号を濃くするために使う。

こはるなぎ

コード

インラインコードは max-width: 40ch のように表示されます。ブロックは：

article {
  max-width: 40ch;
  margin: 0 auto;
}

キーボード

保存は ⌘ + S、送信は ⌘ + Enter。

設計原則

  1. クラスを書かせない。セマンティックに書くだけで整う。
  2. 日本語で読むことを前提に、行長と行間を設計する。
  3. 色は最小限、装飾は罫線と余白だけ。
  4. ダークモードに自動対応。印刷にも対応。
  5. CSS変数で上書き可能。思想は借りつつ、色は自分で。
